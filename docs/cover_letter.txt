Проблеми української освіти:
    1) Брак коштів
    2) Низька (нерівномірно розподілена) якість
    3) Мовні проблеми (етнічні меншини, іноземні студенти)
    4) Великий вік викладачів (пенсія, смерть)

Рішення:
    Переведення лекцій в онлайн


ПЛЮСИ:
Для студентів:
    1. Можливість отримувати лекції від провідних спеціалістів в галузі
        У мене був найкращий в світі лектор по гомогенному каталізу, академік Попов А.Ф.
        Нажаль, з групи його розуміло 2-3 людини. В той же час, студенти з інших навчальних
        закладів не мають можливості прослуховувати його лекції через географічну віддаленість
        (Донецьк)
    2. Більш вільний розклад
        Багато студентів працюють і через це не мають можливості відвідувати весь курс лекцій
    3. Підвищення якості заочного навчання.

Для міністерства:
    1. Вартість лекцій для міносвіти зменшиться. Платиш один раз - маєш лекції завжди
        (можна додати додаткове заохочування лекторів за кількістю просмотрів).
    2. Лекції зберігаються навіть при фізичній відсутності викладача.
    3. Тепер кожному університету більше не потрібно розробляти власне ПО для донесення
        лекцій до заочників. Вони можуть перенаправити ці кошти на більш інтерактивні додатки.
        Або економія.
    3. Підвищення публічності роботи міністерства.
    4. (Можливо) через більшу легкість перекладу
        підвищення популярності української освіти в Україні по всьому світу.
        можливість відтягнути в Україну освіту до рівня бакалавра з Європи.

Для викладачів
    1. Менше часу на повторення одного й того ж (зменшення рутини).
    2. Можливість витрачати час не на говоріння а на покращення лекцій.
    3. Більша аудиторія - більший вплив.
    4. Можливість залишитися в часу (навіть після смерті викладача).
    5. Більш вільний графік роботи.

Для суспільства:
    Якщо ми робимо всі лекції відкритими:
    1. Life-long education (Освіта на протязі життя)
    2. Зменшення напруги в мовній сфері
    3. Можливість освіти у в’язницях


ДОДАТОК (ЯК ВОНО ПРАЦЮЄ)
Воркфлоу з боку викладача:
    0. Зареєструватися (легко - через соціальні мережі)
    1. Залити відео на youtube
    2. Додати питання для підвищення інтерактивності
        Згідно з великою кілкістю дослідженнь (навіть є статті в медіа)
        http://news.bbc.co.uk/2/hi/uk_news/education/8449307.stm
        середній студент може зосереджуватися (без взаємодії) біля 10 хвилин
    3. Опублікувати курс

Воркфлоу з боку студента:
    0. Зареєструватися (легко - через соціальні мережі)
    1. Обрати курс
    2. Переглянути відео
    3. Відповісти на питання


Issues
1. Необхідність інфраструктури
    Рішення: використати датацентри Януковича - молодшого
    + (можливо) гранти на реформи
    + (можливо) гранти від українських меценатів
    + (можливо) громадське фінансування
2. Кошти на розробку
    Рішення: залучити волонтерів з програмістів
    + (можливо) гранти на реформи
    + (можливо) гранти від українських меценатів
    + (можливо) громадське фінансування
3. Незвичність інтерфейсу (комп’ютер) для викладачів старшого віку
    Рішення: залучення студентського активу.
    Інтерфейс максимально спрощено + є відеоінструкції.
4. Зменшення очного спілкування
    Насправді, це не є проблемою, бо за умов великої аудиторії насправді нема особливої
    можливості задавати питання. Також знімається через поступове перенаправлення коштів
    з лекцій на семінарські зайняття.
    + За рахунок більшої публічності лекцій викладач ставитимуться більш відповідально
    до тих лекцій що вони читатимуть. Хороша онлайн-лекція більш ефективна
    за погану офлайн-лекцію.


Пропоную наступні плани впровадження:
1. М’який:
    1.1. Переведення лекцій в онлайн по бажанню викладачів. Лекції в інтернеті
    співіснують з оффлайн-лекціями. Призи за найкращі/найпопулярніші лекції.
    1.2. Викладачі що мають онлайн-лекції перестають читати в оффлайні, час на лекції
    може бути використаний на їх покращення.
    1.3. Переклад лекцій на мови України та світу.
2. Економний:
    2.1. Массове переведення лекцій в онлайн в добровольно-примусовому порядку.
    2.2. Перенаправлення коштів на інші activities (наприклад, семінарські зайняття)


Наприкінці треба зазначити що на даний момент система має базовий функціонал
і потребує допрацювання:
    1. Статистика відповідей для зворотнього зв’язку з викладачем.
    2. Мультилінгвальність (один і той самий курс може викладатися декількома мовами).
    3. Додання мультиваріантних відповідей та відповідей з текстовим вводом.
    4. Спільна робота декількох викладачів та/або перекладачів по курсу.
    5. Версіонування (зберігати попередні редагування).
    6. Логіка перекладання, версіонування перекладів.
